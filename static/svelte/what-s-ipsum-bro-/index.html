Now you're ready to get coding!

Look through each file in the src folder, which is where all the code you write will live.

There's no index.html, instead there's +page.svelte in the routes folder. This file acts as the landing page.

There are several folders inside routes, and each one has its own +page.svelte file. The name of the folder is the name of the web page, and the +page.svelte is that individual web page.

+page.svelte files have a script section at the top for JavaScript, the middle can contain HTML, and the bottom has a style section for CSS. All the JavaScript, HTML, and CSS for a page is stored together, which makes it easier to work on individual chunks of your website.

Header.svelte, Nav.svelte, and Footer.svelte in the lib folder are examples of components. They're reusable chunks of your website that you can use on any +page.svelte by importing them in the script section, and then adding the element in the HTML section.

+page.svelte

<script>
  import Header from '$lib/Header.svelte'
</script>

<Header />

To create the header, we are going to add some style to the header component that's already in the code.

We're going to add two main things:

  • a new font from Google Fonts
  • placement and colour styles

Font

To link to a Google Font that you want to use on your site, we simply add the @import line to the +layout.svelte file, at the top of the style section (where the comments say to add global styles). This makes the font available everywhere on the site.

To get the @import, select the option when you've chosen the font you want.

In your +layout.svelte file, paste the @import line below the comment about global styles.

+layout.svelte

@import url('https://fonts.googleapis.com/css2?family=Patrick+Hand&display=swap');

Now in your Header.svelte file, you can add the colours, spacing, and the font as CSS rules in the style section at the bottom. If there isn't a style section, you can create one simply by adding a style element.

+Header.svelte

<style>
  header {
    background: lavender;
  }

  h1 {
    font-family: 'Patrick Hand', cursive;
    text-align: center;
    margin-top: 100px;
    margin-bottom: 0;
  }
</style>

In the final design, we are going for a two-column layout with text in one column, and the images in the other.

We'll achieve this in 3 steps:

  1. Group the boxes together
  2. Group the text together
  3. Wrap the two groups in a container and make them sit side-by-side

Group the boxes

Put a div element around the as that make up the boxes, and give it the boxes class. We need some layout CSS to get the boxes to sit in a nice 2x2 arrangement, and Flexbox is perfect for this.

Write CSS Flexbox rules for the boxes class that makes the boxes arrange themselves into two rows of two, with even space between them. You can use the rules below, modify them, or create your own.

+page.svelte

/* container for all the boxes */
.boxes {
  display: flex;
  flex-wrap: wrap;
  gap: 25px;
  justify-content: space-between;
  width: 400px;
}

Group the text

Put a div element around the text, and give it the text class, then add a rule to use the new font and reduce the width so that the text isn't so greedy on page space.

Remember to centre the h2 element somehow.

+page.svelte

.text {
  font-family: 'Patrick Hand', cursive;
  width: 30%;
}

Put the groups into columns

Put a div element around the text and boxes elements, and give it the columns class.

In your CSS, create a .columns class and use Flexbox again to get the contents of the columns container to sit side-by-side.

We're also going to tell the contents to vertically align themselves in the middle of the columns container, and divide any remaining horizontal space evenly around themselves.

+page.svelte

.columns {
  display: flex;
  align-items: center;
  justify-content: space-evenly;
  background: lavender;
  padding: 5rem 10rem;
}

Make it mobile friendly

Press Command + Option + I on MacOS or Ctrl + Shift + I on Windows to open the Developer Tools in Chrome and look for the device icon in the top left. Use it to switch between desktop and mobile views.

You'll notice that the mobile version currently has far too much padding to look nice.

Add some rules to put the contents of the columns vertically on mobile, and remove some of the extra space. You can use the rules below, modify them, or create your own.

+page.svelte

/* adjustments for mobile */
@media screen and (max-width: 600px) {
  .columns {
    flex-direction: column;
    padding: 2rem 3rem;
  }
    
  .text {
    width: 100%;
  }
}

The wave graphic at the bottom adds some visual interest and breaks up the page. There are several ways to achieve the effect, but we're going to use CSS generated by a website. The CSS creates the wave effect completely from code, which makes the page very fast to load because it's just a few words, instead of a whole image.

Add a div with the class wave, below your columns (not inside them). This will get transformed into the wave by the CSS.

Paste the CSS below into your style section. It includes some colour for the main element so that the wave will be visible against it as a background.

Don't worry if you don't understand the code that makes the wave - neither do I. It works though.


+page.svelte

main {
  background: lightblue;
  padding-bottom: 3rem;
}

/* wave effect generated at https://css-generators.com/custom-borders/ */
.wave {
  --mask: radial-gradient(52.5px at 50% calc(100% - 74px), #000 99%, #0000 101%) calc(50% - 80px) 0/160px 100%, radial-gradient(52.5px at 50% calc(100% + 34px), #0000 99%, #000 101%) 50% calc(100% - 40px) / 160px 100% repeat-x;
  mask: var(--mask);
  background: lavender;
  height: 70px;
}

  • Motion effects to add interest

A little bit of motion can make your page feel more alive (too much can make people feel motion sick, though). We're going to add two bits of motion to the page:

  • a hover effect on the boxes to help identify them as links
  • a hover effect on the wave for reasons

Scale the boxes on hover

Add the CSS below to transform the scale of the boxes to 10% bigger (1.1 times normal size) when they're hovered over.

.box:hover {
  transform: scale(1.1);
  transition: 0.25s;
}

Add the same transition rule to the box class so that it takes 0.25s to go back to its normal size.

Scale the wave on hover

Add the CSS below to transform the scale of the wave to 10% taller and 50% wider when it's hovered over.

.wave:hover {
  transform: scale(1.1, 1.5);
  transition: 0.5s;
}

Add the same transition rule to the wave class so that it takes 0.5s to go back to its normal size.
