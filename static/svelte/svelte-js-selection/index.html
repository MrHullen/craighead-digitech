Before we get into Svelte's {#if} statements, let's quickly review how if statements work in vanilla JavaScript.

<script>
  let name = 'Alice'

  if (name) {
    alert('Hello, ' + name + '!')
  } else {
    alert('Hello, stranger!')
  }
</script>

In this example, the if statement checks if the variable name is true (has a real value). If it does, it outputs a personalised greeting. If name is false (e.g. an empty string, null, or undefined), it outputs a generic greeting.

In Svelte, you can use {#if} statements to show or not show content based on a variable. Here's the same example from before, but now using the page instead of alert. 

<script>
  let name = 'Alice'
</script>

{#if name}
  <h1>Hello, {name}!</h1>
{:else}
  <h1>Hello, stranger!</h1>
{/if}

Explanation

Script Tag: Define the name variable inside the script tag.

<script>
  let name = 'Alice';
</script>

Conditional Rendering: Use the {#if} block to show different HTML based on the value of the name variable.

{#if name}
  <h1>Hello, {name}!</h1>
{:else}
  <h1>Hello, stranger!</h1>
{/if}
  • {#if name}: This block checks if the name variable has a truthy value.
  • h1 Hello, {name}! /h1 : If name is truthy, this line renders an h1 element with the personalised greeting.
  • {:else}: This block specifies the content to render if the if condition is not met.
  • h1 Hello, stranger! /h1 : If name is falsy, this line renders an h1 element with the generic greeting.
  • {/if}: This closes the if block.

  3. Try changing the value of name to an empty string and see what happens.

In Svelte, you can do comparisons inside {#if} statements, just like with regular if statements. Here's an example where we check if the name variable is equal to a specific value using the == operator.

In this example, we will check if the name variable is equal to the string 'Alice' and render different content based on this condition.

<script>
  let name = 'Alice'
</script>

{#if name == 'Alice'}
  <h1>Welcome back, Alice!</h1>
{:else}
  <h1>Hello, {name}!</h1>
{/if}

Explanation

Script Tag: Define the name variable inside the script tag.

<script>
    let name = 'Alice';
</script>

Conditional Rendering with Comparison: Use the {#if} block with the == operator to compare the name variable to the string 'Alice'.

{#if name == 'Alice'}
    <h1>Welcome back, Alice!</h1>
{:else}
    <h1>Hello, {name}!</h1>
{/if}
  • {#if name == 'Alice'}: This block checks if the name variable is equal to the string 'Alice'.
  • h1 Welcome back, Alice! /h1 : If name is equal to 'Alice', this line shows an h1 element with a personalised welcome message for Alice.
  • {:else}: This block specifies the content to show if the if condition is not met.
  • h1 Hello, {name}! /h1 : If name is not equal to 'Alice', this line shows an h1 element with a generic greeting that includes the value of the name variable.
  • {/if}: This closes the if block.

In Svelte, the {:else if} block allows you to add additional conditions within an {#if} block, similar to else if in vanilla JavaScript. This enables you to check multiple conditions sequentially and render different content based on which condition is met. 

<script>
  let name = 'Alice'
</script>

{#if name == 'Alice'}
  <h1>Welcome back, Alice!</h1>
{:else if name == 'Bob'}
  <h1>Not you again!</h1>
{:else}
  <h1>Hello, {name}!</h1>
{/if}

The {:else if} block is used after an {#if} statement and before an {:else} block. If the initial {#if} condition evaluates to false, the {:else if} condition is checked. If the {:else if} condition evaluates to true, the corresponding block of code is executed. You can include multiple {:else if} blocks to handle various scenarios. If none of the conditions in the {#if} or {:else if} blocks are met, the {:else} block is executed as a fallback.
