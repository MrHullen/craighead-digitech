Checkpoint

You've started a Firebase project and are ready to use its features.

Firebase includes a feature called Cloud Firestore, which is their database. A database holds data for your web app and will send it to your app when you need it. Keeping the data separate helps keep everything manageable, flexible, and robust.

  1. Click on the Build menu dropdown and select Firestore Database.
  2. Click Create database to create the database for this project.

  1. Click Next on the Production mode page, and Enable on the Location page.

Checkpoint

You've set up a Firestore database and are ready to add some data to it.

Create a collection

A Collection is the name for a group of data items. Our practice data will be Barry Batman and Thomas Tornado, so the Collection of these items will be called people.

  1. Create a collection by clicking Start collection.

  1. Give your Collection the ID people and click Next.

Add the first document

Documents are the objects that get stored in the database. Each document can have fields inside it that represent data about the object. For our example, each document will represent a different person.

  1. The first field can be firstName with the type String and value Barry
  2. The second field can be lastName with the type String and value Batman
  3. The third field can be age with the type number and value 18
  4. The document ID can be the same as the firstName for now, i.e. Barry

Tip: Using the name as an ID isn't the best practice because you can't guarantee that two people won't have the same name. You should leave the ID on auto-generated. We've used the name here for clarity.

  1. Hit Save and you'll see your first document representing the person, Barry Batman.

Add another document

Create a new person called Thomas Tornado who is 19 years old.

Allow access

Databases often store information that you want to control access to, like people's email addresses or even passwords. Firebase lets you do that with rules.

When you first create your database, by default no one can access anything. This is to make sure you don't accidentally share something you don't want to.

Your database doesn't store anything sensitive yet, so we'll change the rules so that anyone can read or write information to the collections.

  1. Go to the Rules tab of your database.
  2. Change line 5 so that it says true instead of false.
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}

  1. Click Publish to save the rule change you've made.

This line creates a constant variable and makes it store all of the documents from the database. It also has the await keyword, which is linked to the async keyword from before. When you use the await keyword, the app will wait until the data arrives before moving on with the rest of the function.
