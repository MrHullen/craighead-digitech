Local

  3. Click the Open in Visual Studio Code button (or press Command + Shift + A on MacOS or Ctrl + Shift + A on Windows)

Codespaces

  1. Login to GitHub.com and find your Introduction repository
  2. Click on the green Code button, change the tab to Codespaces, and choose the workspace under On current branch (it will have a randomly generated name)

Start your dev server

  1. Open a new Terminal by clicking New Terminal in the Terminal menu, or pressing Ctrl + `
  2. Start the server on your laptop by entering the command npm run dev
  3. Open the link that it creates to view your site

The server will update the site in your browser every time you save a file, so that you can easily see the changes as you make them.

Creating new pages in Svelte is as easy as creating a new file in your routes folder. We're going to create a second page so that you can practice reusing components, and learn how to create a reusable nav menu.

  1. Create a new page by creating a new folder in your routes folder, and call it about-me
  2. Create a new file insite about-me called +page.svelte
  3. Import and add your Header and Footer components to the page
  4. Add in a main section with some information about yourself
<script>
  import ComponentName from 'ComponentName.svelte'
</script>

<ComponentName />

Reminder

Components are reusable chunks of code that sit in their own file. You can import and use those chunks anywhere on your website.

When you update the component, anywhere it's used on your site will be updated as well, which makes it easy to iterate and improve.

Commit point

Navigation menus can quickly become unwieldy when your site gets large enough - update one page name and you have to go through every nav menu and update them accordingly.

In modern websites, developers create one reusable nav menu.

  1. Create a new component by creating a new file in your lib folder, and call it Nav.svelte
  2. Copy and paste the code below into your new component
<nav>
  <a href="/"> Home </a>
  <a href="/about-me"> About Me </a>
</nav>
  1. Import and add your Nav component to both pages

Links

You'll notice that your links start with / and then the name of your page, or just / for your home page. You might also notice that it does not include the file extension .svelte.

This is because Svelte will handle most of the details of the navigation in the background, so that you can focus on more important things.

Customise

Change your nav to personalise your web page.

  • Add a style element to your component and write your own CSS
  • Make sure you have a hover effect so that the user knows it's a link
  • Try some of these:

text-align: center;

text-decoration: line-through;

text-transform: uppercase;

text-shadow: 2px 2px orange;
