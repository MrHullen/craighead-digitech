Discussion

Why does the file have to be called index.html?

Now you're ready to get coding!

Copy and paste this code to your index.html file:

<h1>
  Introduce Yourself
</h1>
    
<h2>
  About Me
</h2>

What is all that?

h1 stands for heading 1, which is the biggest heading. It shows where the heading starts.

/h1 has a slash at the start. This shows where the heading ends.

h1 is a heading opening tag and /h1 is a heading closing tag.

Together, they make a heading element.

New vocabulary: opening tag, closing tag, element

Customise

Change your two heading elements to personalise your web page.

  • Change the text inside your heading 1 element so that it has your name in it
  • Change the text inside your heading 2 element so that it has your year level in it

Commit point

Now that you've added a feature to your website, you should commit it to your GitHub repository! Committing creates a full backup and a snapshot of the code at this point.

Open GitHub Desktop and check that the Current Repository in the top left is your project, and briefly review the changes you've made. Green indicates code that you've added since your last commit, and red is code you've deleted.

When you're happy, write a short (2 - 5 words) Summary of what you added in the bottom left. In this case, your message could be Added headings.

Copy and paste this code after your /h2 closing tag in your index.html file:

<p>
  A paragraph about me.
</p>

p stands for paragraph, which is a basic block of text. This opening tag shows where the paragraph starts.

/p has a slash at the start. This closing tag shows where the paragraph ends.

The p and /p tags together make a paragraph element.

Customise

Change your paragraph element to personalise your web page.

  • Change the text inside your paragraph element so that it has a little about you in it

Copy and paste this code after your /p closing tag in your index.html file:

<p>
  My interests include:
</p>

<ul>
  <li> Thing 1 </li>
  <li> Thing 2 </li>
  <li> Thing 3 </li>
</ul>

The paragraph element should be familiar from the last part. The ul element underneath might be a bit intimidating though. Let's look at it bit-by-bit.

ul stands for unordered list, which is a list of items with bullet points. This opening tag shows where the list starts. At the end, /ul has a slash. This closing tag shows where the list ends. The ul and /ul tags together make a list element.

A list wouldn't be much without list items. To create this there is more code inside the list element. Each list item has its own tag, li for the opening tag and /li for the closing tag. Together, they make a list item element.

This unordered list element has 3 list item elements in it.

Customise

Change your list item elements to personalise your web page.

  • Change the text inside each list item element so that it has an interest of hobby of yours in it
  • Add two more list item elements with more interests, hobbies, sports, or anything!

Tip

If your code gets a bit ugly and isn't lined up nicely, you can press Option + Shift + F (MacOS) or Alt + Shift + F (Windows) to format it perfectly.

Formatting correctly helps you and other developers read your code more easily. You can clearly see where HTML elements start and finish because their opening tags and closing tags are lined up.

Along the way, you've learned about these parts of HTML.

Code

➕ h1 for main headings

➕ h2 for subheadings

➕ p for paragraphs

➕ ul for unordered lists (bullet points)

➕ li for list items inside an unordered list

Vocabulary

➕ tag - a single piece of HTML code inside pointy brackets

➕ element - a chunk of the webpage, such as a heading or paragraph
